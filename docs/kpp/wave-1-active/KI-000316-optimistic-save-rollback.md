# KI-000316 · Save Reliability — No Silent Data Loss (सेव की विश्वसनीयता)

**Definition:** Save reliability means that when you save an entry, it is genuinely stored to the cloud — and if a save fails, the software tells you and does not pretend it succeeded, so you never lose work without knowing.

**Plain-language explanation:** When you save, it really saves online — and if it can't, you get a clear warning instead of a false "saved". No quiet data loss.

**Hindi explanation (आसान हिंदी):** सेव की विश्वसनीयता का मतलब है — जब आप कोई एंट्री सेव करते हैं तो वह सचमुच क्लाउड में संग्रहीत होती है; और अगर सेव असफल हो तो सॉफ्टवेयर साफ़-साफ़ बताता है, झूठा "सेव हो गया" नहीं दिखाता। इससे आपकी मेहनत चुपचाप कभी नहीं खोती।

**English explanation:** A data-integrity guarantee that saves are confirmed to the cloud, and failures are surfaced (not hidden) — so the screen never disagrees with what is actually stored.

**Why it matters:** The worst accounting fear is "it looked saved but wasn't, and on refresh the entry was gone." Reliable, honest saving protects real work — a core SahakarLekha principle.

**Common misconceptions:**
- ❌ "If the screen shows it, it must be saved." → Only true if the save was actually confirmed; good software verifies and warns on failure.
- ❌ "Cloud apps can lose entries silently." → A well-built one surfaces failures instead of hiding them.
