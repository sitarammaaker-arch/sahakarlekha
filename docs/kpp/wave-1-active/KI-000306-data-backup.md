# KI-000306 · Data Backup (डेटा बैकअप)

**Definition:** Data backup is keeping a safe extra copy of the society's accounting data so it can be recovered if the original is lost, damaged, or accidentally changed.

**Plain-language explanation:** A backup is a spare copy of your data kept safe, so nothing is lost if something goes wrong.

**Hindi explanation (आसान हिंदी):** डेटा बैकअप यानी समिति के हिसाब-डेटा की एक सुरक्षित अतिरिक्त प्रति रखना, ताकि अगर मूल डेटा खो जाए, खराब हो जाए या गलती से बदल जाए, तो उसे वापस पाया जा सके। आसान शब्दों में — "ज़रूरत पड़ने पर बहाल करने के लिए डेटा की सुरक्षित नकल"।

**English explanation:** A protected duplicate of the society's data enabling recovery after loss, corruption, or error; in cloud software, backups are handled automatically on secure servers.

**Why it matters:** Societies' records are irreplaceable. Backups protect against data loss — a major reason cloud accounting is safer than a single local computer.

**Common misconceptions:**
- ❌ "Backup is only needed when something breaks." → Backups must exist *before* a problem, not after.
- ❌ "Cloud means I never need to think about backup." → Cloud automates it, but understanding restore (recovery) still matters.
