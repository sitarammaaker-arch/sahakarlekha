# KI-000310 · User Role (उपयोगकर्ता भूमिका)

**Definition:** A user role is a named set of access rights in the software (such as Admin, Accountant, or Data-entry) assigned to each user — controlling what they can see and do, so responsibilities and access match each person's job.

**Plain-language explanation:** A role decides what a user can do in the app — an admin can do everything, a data-entry user only what they need.

**Hindi explanation (आसान हिंदी):** उपयोगकर्ता भूमिका सॉफ्टवेयर में पहुँच-अधिकारों का एक नामित समूह है (जैसे Admin, Accountant, Data-entry) जो हर उपयोगकर्ता को दिया जाता है। यह तय करता है कि वह क्या देख व कर सकता है — ताकि हर व्यक्ति की पहुँच उसके काम के अनुसार रहे।

**English explanation:** A predefined access level assigned per user, enforcing separation of duties and least-privilege access.

**Why it matters:** Roles keep data safe and controls clean — not everyone should be able to edit or delete everything. Auditors value role-based access.

**Common misconceptions:**
- ❌ "Everyone should be admin for convenience." → That removes control; assign the least access each role needs.
- ❌ "Roles and individual users are the same." → A role is the access level; users are assigned a role.
