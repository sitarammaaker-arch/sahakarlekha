# KI-000311 · User Permission (उपयोगकर्ता अनुमति)

**Definition:** A user permission is a specific right to perform an action in the software — such as create a voucher, view a report, approve, or delete — granted (usually via a role) to control exactly what each user can do.

**Plain-language explanation:** A permission is a single "you can / cannot do this" rule — like "can create vouchers" or "can view reports" — set for each user.

**Hindi explanation (आसान हिंदी):** उपयोगकर्ता अनुमति किसी खास काम को करने का अधिकार है — जैसे वाउचर बनाना, रिपोर्ट देखना, स्वीकृति देना, या मिटाना। यह (आम तौर पर भूमिका के ज़रिए) दिया जाता है ताकि हर उपयोगकर्ता ठीक उतना ही कर सके जितना ज़रूरी हो।

**English explanation:** The granular right behind a role; permissions enforce who can do what, protecting data and controls.

**Why it matters:** Right permissions prevent accidental or unauthorised changes (e.g. only certain users can delete), keeping the books safe and audit-clean.

**Common misconceptions:**
- ❌ "Permission and role are identical." → A role is a bundle of permissions.
- ❌ "Everyone needs delete rights." → Sensitive rights (delete, approve) should be limited.
