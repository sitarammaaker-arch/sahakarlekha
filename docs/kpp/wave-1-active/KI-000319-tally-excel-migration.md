# KI-000319 · Migrating from Tally / Excel (Tally / Excel से माइग्रेशन)

**Definition:** Migration from Tally/Excel is the one-time move of a society's existing records (masters, opening balances, and history) from a previous tool into the new software — typically via export from the old system and a bulk import into the new one.

**Plain-language explanation:** It is shifting your existing data from Tally or Excel into the new software once, so you continue from where you were instead of starting blank.

**Hindi explanation (आसान हिंदी):** Tally/Excel से माइग्रेशन यानी समिति के मौजूदा रिकॉर्ड (मास्टर, प्रारंभिक शेष, और इतिहास) को पुराने उपकरण से नए सॉफ्टवेयर में एक बार स्थानांतरित करना — आम तौर पर पुराने सिस्टम से export करके नए में bulk import द्वारा। इससे आप शून्य से नहीं, बल्कि जहाँ थे वहीं से आगे बढ़ते हैं।

**English explanation:** A one-time data move (export → import) from a prior tool into SahakarLekha, so the society keeps its history and continues smoothly.

**Why it matters:** Fear of "losing our old data" blocks many societies from switching. A clear migration path removes that fear.

**Common misconceptions:**
- ❌ "Switching means losing old records." → Migration brings masters, opening balances, and history across.
- ❌ "Migration must be perfect in one shot." → Core data first (masters + opening balances); details can follow.
