# 07 — State-wise Research Registry

> A research **framework for every Indian State and Union Territory**, because cooperative law is
> primarily *state* law. Each row is a **research stub** to be filled and validated — **not** a
> statement of current law.
>
> **Important:** the "Applicable Act" column holds a **research lead** (commonly-cited statute name) that
> **must be confirmed against the current amended gazette text** before any use. All act titles, years,
> and provisions are `⚠️ NEV` until E3 ([01](01-research-methodology.md)/[06](06-law-and-compliance.md)). Language is
> the only field safe to treat as factual. **Single-state societies follow state law; multi-state →
> MSCS Act (CENTRAL).**

**Per-state record:** Applicable Act (lead, `⚠️ confirm`) · Registrar (RCS) · Major coop types (lead) ·
Primary language(s) · Special compliance areas (research) · Research status · Source ids to instantiate.
**Status baseline:** all `open` at v1.0. Source ids to create: `SRC-ST-ACT-{XX}`, `SRC-ST-RULES-{XX}`,
`SRC-ST-RCS-{XX}`, `SRC-ST-AUDITMAN-{XX}` ([02 G2](02-source-registry.md)).

## Research workflow per state (to raise status open → validated)
1. Instantiate the 4 `SRC-ST-*-{XX}` source records ([02 G2](02-source-registry.md)).
2. Confirm **current amended Act + Rules title/year** from the official gazette/RCS site → E2. `⚠️`
3. Capture: society classes recognised, audit authority + classification, statutory registers, return forms + deadlines, reserve %, dividend cap, election rules, member-list (Form-1) format.
4. Tag jurisdiction `STATE:{XX}`; note where it diverges from neighbours.
5. **SME sign-off** → E3; stamp `as_of` + `recheck_on`.
6. Feed [SCOS D24/D28](../scos/03-topic-registry.md) state pages — **only** publish a state page once its
   record carries at least one validated state-specific fact (thin-content guard).
