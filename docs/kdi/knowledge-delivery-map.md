> knowledge show up" matrix. Targets reference **existing** surfaces only (no new architecture); each
> KI's own suggested surfaces (from its [KPP record](../kpp/wave-1-active/00-index.md)) seed the row.

AI=Ask-AI grounding · Mod=Software module · Dl=Download · Vd=Video · Em=Email · IL=Internal links · Se=Search · Nv=Navigation/footer
**Legend:** ✓ = applicable destination · — = not applicable · all 50 get **AI + Search + Internal-links** by default.

## Delivery rollup
- **Glossary:** 42/50 KIs (all term/concept KIs) → the single highest-leverage surface.
- **FAQ:** ~22/50 (questions + high-traffic concepts).
- **Context help (in-module):** ~24/50 (every KI with a `related_module`).
- **Ask-AI grounding + Search + Internal links:** **50/50** (universal).
- **Navigation/footer:** ~9 (top product/marketing KIs).
- **Email/onboarding:** ~10 (onboarding + adoption KIs).

> **Rule:** each delivery target reuses the KI's existing `internal_links`, `related_module`, and
> `suggested_*` fields from its KPP record — KDI just routes them. Where a destination needs the
> *definition text*, it is **generated from the KI** (single source of truth), never re-written.
