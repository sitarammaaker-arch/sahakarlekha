# Chapter 2 — Registration, Society Setup & Roles

> **After this chapter you will be able to:** create a new society; set society details and the financial year; understand roles and Maker-Checker; read the Dashboard.

## 2.1 SahakarLekha at a glance
A free, cloud-based accounting software built specially for Indian cooperative societies. Open sahakarlekha.com in a browser — nothing to install.

## 2.2 Registering a new society
sahakarlekha.com → "New society? Register" → name, registration number, state/district, mobile, email → a strong password → Register. You become the **Admin**.

> 💡 One society = one separate, secure world of data (society-level security/RLS).

## 2.3 Login & password
"Forgot password?" → a reset link on email (check spam too). After reset, both routes work with the same new password.

## 2.4 Roles & Maker-Checker

| Role | Can do | For whom |
|---|---|---|
| Admin | Everything | Secretary/Manager |
| Accountant | Entries, reports | Accountant/Clerk |
| Viewer | View only | Member/Auditor |

**Maker-Checker:** one person creates an entry (Maker), another approves it (Checker). In a voucher, choose "Send for approval".

## 2.5 Society setup

| Field | Why it matters |
|---|---|
| Society name, registration | Report/PDF heading |
| Financial year (e.g. 2026-27) | All reports are for this period |
| Signatories | Signature block below reports |

> ⚠️ Choose the financial year carefully — a wrong year = empty/wrong reports.

## 2.6 Dashboard
Cash/bank balance, total members, this month’s sales/purchases, outstanding dues, recent vouchers — the society’s health at a glance.

## ⚠️ Common mistakes

| Mistake | Avoid by |
|---|---|
| Wrong financial year | Check the setup |
| Everyone an Admin | Assign roles thoughtfully |
| One shared login | A separate user per person |

## 🔍 Audit note
"User roles are defined; maker-checker control is in place — internal control is satisfactory."

## 📘 Cooperative standard
Each society’s accounting year runs 1 April to 31 March; the setup should match.

## ✅ Worked example
Rania Society: financial year 2026-27; Ram = Admin, Shyam = Accountant. Register → setup → add users → assign roles.

## 📚 Case study
One shared Admin login for everyone → no way to know who made a wrong entry. Separate users + Maker-Checker let mistakes be caught at once.

## ❓ FAQ
**Q. Several societies from one mobile?** Yes, each with separate data.
**Q. No internet?** It is cloud-based; you can view report PDFs offline.
